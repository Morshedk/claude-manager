How should multi-line input behave when you press Send?

The current bug: bracket-paste wrapping tells the shell "this is a paste, don't execute." The fix removes it — but we need to decide what happens when you send multiple lines at once.

Option A

Send all lines, execute as a block

The entire textarea content is sent to the terminal as-is with a final Enter. Multi-line scripts (heredocs, for-loops) arrive intact and execute together.

User types: for i in 1 2 3\ndo echo $i\ndone
↓ press Send
Terminal receives full block + ↵ → executes
↓
1
2
3

Best for scripting. Multiline bash constructs work. Single commands also work fine.

Option B

Send each line separately

Each newline in the textarea becomes a separate command execution. Paste a list of commands — each runs in order. Simpler mental model for command queues.

User types: cd /tmp\nls\npwd
↓ press Send
Line 1: cd /tmp ↵ → executes
↓ 200ms delay
Line 2: ls ↵ → executes
↓ 200ms delay
Line 3: pwd ↵ → executes

Intuitive for command lists. Breaks multiline bash constructs (for-loops, heredocs).

Pick one — or tell me in the terminal. (For single-line commands, both options behave identically.)
