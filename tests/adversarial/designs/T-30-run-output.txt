=== ROOT CAUSE OF INITIAL FAILURE ===
  The test used: execSync(`tmux send-keys -t ... "echo PIDMARKER=$$" Enter`)
  $$ is expanded by execSync's sh -c shell (a new process each time), NOT by
  the bash inside the tmux pane. So each invocation captured a different PID —
  the PID of the ephemeral sh subprocess, not the stable bash inside tmux.

  Fix: use `tmux display-message -p '#{pane_pid}'` which queries the tmux server
  directly for the PID of the process running inside the pane. No shell expansion.
