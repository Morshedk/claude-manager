# T-01 Test Run Output

## Attempt 1 (FAILED — test infrastructure issue)
Timeout: 120000ms
Result: Test timed out at 120s

Root cause: `deleteSession()` called a non-existent REST endpoint
(`DELETE /api/sessions/:id`). The server has no such endpoint — session
deletion is WS-only. Because deletes failed silently, sessions were not
cleaned up between iterations, and the DOM `waitForFunction` for cards
clearing never resolved. After 1 iteration the test timed out.

Additionally, SIGTERM appeared in server output mid-test, which caused
subsequent API calls to fail silently, leading to the test hanging until
timeout.

SIGTERM post-mortem: The SIGTERM in attempt 1 and 2 was our own afterAll
cleanup firing when the test timed out. In attempt 3 (successful run to
completion), the SIGTERM appears at the very end after all 5 iterations
completed normally — confirming it was afterAll cleanup, not a bug.

## Attempt 2 (FAILED — same infrastructure issue, different timeout)
Timeout: 300000ms
Result: Test timed out at 300s for the same reason (broken REST delete endpoint)

## Attempt 3 (COMPLETED — test infrastructure fixed)
Timeout: 300000ms
Result: Test RAN TO COMPLETION with 5/5 iterations all reporting GENUINE FAIL

Fix applied: Replaced `deleteSession()` (REST) with `deleteSessionViaWS()`
(WebSocket `session:delete` message). Sessions now properly deleted between
iterations.
