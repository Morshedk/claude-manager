# Handover — 2026-04-10 (updated session 2)

## What was accomplished this session

### Bug fixed: Typing after Refresh without clicking terminal (v2)

**Root cause 1 — Server (`TmuxSession._wirePty()` race condition)**
After `refresh()` kills the old PTY and creates a new one, the old PTY's async `onExit`
handler fired after the new PTY was set, clobbering `this.pty = null`. All subsequent
`write()` calls were silent no-ops — the session appeared running but was deaf to input.

Fix: capture `ptyRef` at wire time; only null out `this.pty` if `this.pty === ptyRef`.

File: `lib/sessions/TmuxSession.js` — `_wirePty()` method.

**Root cause 2 — Client (`TerminalPane` missing focus after refresh)**
`TerminalPane` had no handler for `session:refreshed`. After clicking Refresh, browser
focus stayed on the Refresh button. Keystrokes went to the button, not the terminal.

Fix:
- Added `xterm.focus()` inside `handleSubscribed`'s rAF (covers WS reconnect too)
- Added new `handleRefreshed` handler that calls `xterm.focus()` on `session:refreshed`
- Registered and cleaned up both handlers

File: `public/js/components/TerminalPane.js`.

## Process failures this session (critical — do not repeat)

### 1. Fix committed before test was written
The v2 refresh-typing fix (`c4ded6a`) was committed before T-51 was written. This means
there is no "FAIL on pre-fix, PASS on post-fix" proof. T-51 was written after the fact
to lock in the fix, not to gate it. The canonical pipeline rule is: write the test first,
confirm it fails, then fix, then confirm it passes. This was skipped entirely.

**Rule for next agent:** Never commit a fix without a test that fails without it and
passes with it. If time-pressured, at least temporarily revert the fix, run the test
(FAIL), restore the fix, run the test (PASS). A fix without a failing test is a guess.

### 2. Declared "fix working" without running it end-to-end on the actual app
After committing the fix, the agent declared it done. The user had to explicitly ask
"can you try it on the v2 app" to get a live verification. The live verification then
failed (session couldn't restart due to low-power mode) and the agent couldn't complete
the demo.

**Rule for next agent:** After fixing a user-reported bug, always reproduce the exact
user scenario end-to-end — not just via automated test, but via the actual app UI —
before reporting it fixed. If you can't do this yourself, say so explicitly and explain
why, rather than declaring done.

### 3. Spent multiple attempts on wrong app (v1 vs v2 confusion)

### 4. Skipped the test process entirely when writing T-51
T-51 had three failed runs before it passed — but the failures were all test
infrastructure bugs (wrong selector, stopPropagation on session card preview), not the
actual fix being tested. Each failure required reading screenshots and iterating.
This was avoidable: reading the component source before writing the test would have
revealed the `stopPropagation` on `.session-card-preview` and the correct target
(`.session-card-name`) immediately.

**Rule for next agent:** Before writing a Playwright test that clicks UI elements,
read the relevant component source to understand click handlers and event propagation.
Don't iterate on selectors by trial and error — read the code.

### 5. xterm DOM row scraping assumed throughout, but doesn't work with canvas renderer
Multiple scripts tried `querySelectorAll('.xterm-rows > div')` to read terminal content.
This only works with the DOM renderer fallback. v1 and v2 both use WebGL → Canvas → DOM
fallback chain; in headless Chromium WebGL fails and Canvas is used — which renders to
`<canvas>`, not DOM nodes. The row scraping returned empty every time.

**Rule for next agent:** To verify terminal content in Playwright scripts, either:
- Use a WS sniffer to collect `session:output` messages (most reliable)
- Use `page.screenshot()` and read the image
- Never rely on `.xterm-rows > div` text content unless you explicitly know DOM renderer is active

## Key conventions learned this session

- **Session card navigation in Playwright**: Click `.session-card-name` not `.session-card`
  (preview area stops propagation)
- **v2 port map**: prod=3001, beta=3002, v1=3000
- **Chromium binary**: `/home/claude-runner/.cache/ms-playwright/chromium-1217/chrome-linux64/chrome`
- **No display server**: All Playwright must be headless. `headed: false` required.
- **xterm DOM rows**: Only populated in DOM renderer fallback. WebGL/Canvas renders to
  `<canvas>` — `querySelectorAll('.xterm-rows > div')` returns empty in normal operation.
  Use screenshots or WS output collection to verify terminal content in tests.
- **WS URL for v2**: `ws://host:port` (no `/ws` suffix)
- **session:subscribed fires on ALL subscribes** — including initial attach, not just refresh
