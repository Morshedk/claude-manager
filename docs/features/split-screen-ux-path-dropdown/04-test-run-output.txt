  - #sidebar-resize found, cursor: col-resize ✓
  - Sidebar drag right 80px → width increased 228→308 (delta=80) — PASS
  - #resize-sessions found in DOM ✓
  - Sessions area drag down 60px → height unchanged 169→169 (delta=0) — FAIL

Root cause identified:
  In App.js, the useEffect that wires up resize handles runs once with [] deps at initial mount.
  At mount time, no project is selected, so ProjectDetail renders the empty state —
  #resize-sessions and #sessions-area do NOT exist in the DOM yet.
  getElementById returns null for both, so cleanupSessions = null and no listener is attached.
  When the user selects a project, ProjectDetail renders #resize-sessions and #sessions-area,
  but the useEffect has already run and will not re-run ([] deps = runs once only).

  The sidebar handle DOES work because #sidebar-resize and #project-sidebar are always in
  the DOM (they are in the App root, not inside conditional ProjectDetail).

  This is a real implementation gap: the sessions area vertical resize handle is never wired up.

Fix applied: None — assertion not watered down. This is a real bug in the implementation.
